# 总表数据入库方案（Parquet，落地对齐版）

## 版本信息

- **版本**：2026-03（落地对齐版）
- **适用范围**：总表（及江苏补源）15 分钟粒度（96 点/日）数据的入库与建模准备。
- **核心目标**（仅三条）：  
  1. **主库可连续使用**：入库后主特征时序连续、可训练、可回测。  
  2. **无效时段与无效特征不混入主表**：把“有行但整列无值”“字段起始过晚”“口径不一致”显式剔除。  
  3. **少量缺失可修复，长段缺失不硬补**：短缺口规则化修复，长缺口走替代源、降级字段或切样本窗。

## 一、总体入库原则

### 1.1 入库目标：两层结构

建议把“总表数据入库”分成两层，各司其职：

| 层 | 名称 | 职责 | 产出 |
|----|------|------|------|
| **L1** | 原始标准化层（raw_normalized） | 保留原始来源信息，做字段标准化、时间标准化、缺失标注，**不强行补值** | 可追溯、可审计 |
| **L2** | 建模可用层（feature_ready） | 在 L1 基础上做字段筛选、缺失修复、替代源拼接、可用区间裁剪 | 可训练、可回测 |

原则：**原始数据不丢，模型特征要严格筛**。

### 1.2 主源与补源原则

- **主源**：`总表/`
- **补源**：`江苏/电价数据/`，仅用于三类情况：  
  - 总表不覆盖的时段（如 2026-01-27～2026-02-28）；  
  - 总表字段整段无效或值级缺失严重、且江苏有同口径字段；  
  - 短缺口修复时的优先替代源。

**禁止**：把总表和江苏直接混成“一个无来源标签的字段”。必须保留：

- `source_system`
- `source_file` / `source_sheet` / `source_column`
- `fill_method`
- `data_quality_flag`（或 `quality_flag`）

### 1.3 缺失处理总原则（三类区分）

| 类型 | 定义 | 处理优先级 |
|------|------|------------|
| **A. 行级缺失** | 某日 96 点整天没有记录行 | 江苏同口径替代 → 邻近日插值/模板修复（仅非价格、缺口很短）→ 无法修复则该日从该特征集剔除 |
| **B. 值级缺失** | 有 96 行但关键列整列 NaN | 按“整天不可用”处理，同 A |
| **C. 长段值缺失 / 晚起始** | 如 storage_actual 起于 2025-12-10、coal_actual 起于 2025-12-23 | **不按缺失去补**。认定为“字段尚未开始稳定提供”：不进入早期样本窗，或在该字段维度置为“未启用” |

**关键**：长段 NaN 不是“可补缺失”，而是“字段未生效”。

> 当前实现补充：已完整支持 A/B/C 的识别与打标（以 `quality_flag` 体现），但“自动修复链路”仍为待增强能力。

## 二、库表设计

### 2.1 L1 原始标准化明细表：power_market_feature_raw（或 dwd_timeseries_15m）

一条记录 = 一个时间点、一个字段、一个区域、一个值。不追求宽表，追求标准化与可审计。

建议字段：

| 字段 | 说明 |
|------|------|
| `ts` | 标准时间戳（datetime64） |
| `trade_date` | 交易日期 |
| `hh_index` | 1～96，当日第几个 15min |
| `phase` | 事前 / 事后 |
| `region` | 汇总(all) / 江南(jn) / 江北(jb) / 华东 |
| `feature_code` | 统一字段编码（见 七） |
| `feature_name_cn` | 中文规范名 |
| `value` | 数值 |
| `unit` | MW / 元/MWh 等 |
| `source_system` | 总表 / 江苏 |
| `source_path` / `source_file` / `source_sheet` / `source_column` | 追溯 |
| `time_source` | 表内日期+时间 |
| `is_missing_row` | 是否因缺行导致该点缺失 |
| `is_missing_value` | 是否值级缺失 |
| `is_filled` | 是否经补全 |
| `fill_method` | 补全方式（见 六） |
| `quality_flag` | 质量等级 0～5（见 六） |
| `etl_batch_id` / `created_at` | 批次与创建时间 |

与现有 `warehouse/dwd/dwd_timeseries_15m.parquet` 对齐时：`ts`↔timestamp，`feature_code`↔metric_id，`region`↔region_id；在此基础上增加 `trade_date`、`hh_index`、`is_missing_*`、`quality_flag` 等。

### 2.2 字段注册表：power_market_feature_registry（或 dim_metric + 扩展）

每个字段一行，定义业务属性与可用区间。

建议字段：

| 字段 | 说明 |
|------|------|
| `feature_code` | 主键 |
| `feature_name_cn` / `feature_name_raw` | 中文名 / 原始列名 |
| `phase` | 事前/事后 |
| `region_level` | 汇总/分区 |
| `unit` | 单位 |
| `main_source` / `backup_source` | 主源(总表)、补源(江苏) |
| `usable_stage` | 日前可用 / 实时滚动 / 仅事后分析 |
| `expected_points_per_day` | 96 |
| `coverage_start_date` / `coverage_end_date` | 原始覆盖起止 |
| **`valid_start_date`** / **`valid_end_date`** | **业务有效起止（晚起始字段必填）** |
| `allow_fill` | Y/N |
| `fill_strategy` | 同口径江苏 / 插值 / 模板 / 不补 |
| `is_model_default_enabled` | 是否默认进主建模库 |
| `remarks` | 备注 |

**必须显式写入的晚起始**：

- `storage_actual`：`valid_start_date = 2025-12-10`
- `coal_actual`：`valid_start_date = 2025-12-23`

### 2.3 L2 建模宽表：power_market_feature_ready_wide

一条记录 = 一个时间点的一整行特征，供模型直接使用。

- 示例字段：`ts`、`trade_date`、`hh_index`、`da_price_south`、`da_price_north`、`rt_price_south_final`、`rt_price_north_final`、`load_actual`、`import_actual`、`pv_actual`、`wind_actual`、`gas_actual`、`reserve_pos`、`reserve_neg`、…
- **每个数值字段配套质量列**，例如：`da_price_south_qflag`、`load_actual_qflag`、`pv_actual_qflag`。  
  训练时可按 `qflag < 4` 过滤或把 `qflag` 作为辅助输入；**不建议只存值不存质量标志**。

产出路径建议：`warehouse/feature_ready/` 下按版本分目录（**V0**、V1/V2/V3），见 九。

## 三、字段分层：哪些进主库，哪些进扩展库

### 3.1 A 层主特征：默认纳入主建模库

覆盖较早、有效性高、连续性最好，作为主数据集的骨架。

- **事后主干**：实际受电 `import_actual`、实际系统负荷 `load_actual`、实际燃机 `gas_actual`、实际光伏 `pv_actual`、实际风力 `wind_actual`
- **事前主干**：受电计划、短期系统负荷预测、正备用、负备用

### 3.2 B 层条件特征：满足起始覆盖后再纳入

- **储能实际** `storage_actual`、**煤电实际** `coal_actual`

处理原则：

- **不进入 2025-12-10 前的联合特征主窗**。
- 若模型必须使用这两类字段，则样本起点至少切到：含储能 2025-12-10、含煤电 2025-12-23。
- 若不强依赖，可做“扩展特征版本”而非主版本。

建议建立两个版本的数据集（见 九）：

- **基础版主样本窗**：不含储能/煤电，起点更早。
- **增强版样本窗**：含储能/煤电，起点 2025-12-23。

### 3.3 C 层禁入主建模库字段

以下**不**默认进入训练特征：

- 事前储能/煤电中的**预发布列**（2025 年大段整列 NaN）
- 与目标时点存在**未来泄露风险**的事后字段
- **2026-01-01～01-26** 总表出清价格整列 NaN 的字段（若无补源修复）

可保留在 raw 层或 analysis 层，但**不默认启用**。

## 四、时间窗裁剪方案（四套标准样本窗）

### 4.1 【优先级最高】方案 0：主窗固定范围（2025-09-01～2025-12-31）

- **时间范围**：**2025-09-01 至 2025-12-31**（闭区间，仅使用该范围内数据）。
- **目标**：避免 8 月成片缺失与 2026 年 1 月出清整段空，主库默认采用此窗，保证连续可用、可训练、可回测。
- **建议纳入字段**：与方案 A 一致——受电实际、负荷实际、燃机实际、光伏实际、风力实际、受电计划、负荷预测、正负备用；若含出清价格，则仅使用该窗内已有有效值的日期（自然避开 2026-01 段）。
- **窗内仍需剔除的日期**（在 9～12 月内）：  
  - 2025-11-16、2025-11-23（正负备用缺行，可补全或标缺失）  
  - 其余 8 月缺日（08-11、08-12～14）与 2026-01 缺日（01-12、01-26）因不在本窗范围内，**无需处理**。

**优先级**：**最高**。建议主训练、主回测、主审计默认先采用本窗；其余方案作为扩展或任务专用。

### 4.2 方案 A：主生产样本窗（最大覆盖）

- **目标**：最大化连续有效性，大多数特征可用。
- **建议纳入字段**：受电实际、负荷实际、燃机实际、光伏实际、风力实际、受电计划、负荷预测、正负备用。
- **建议剔除日期**：  
  - 2025-08-11  
  - 2025-08-12～2025-08-14  
  - 2025-11-16、2025-11-23  
  - 2026-01-12  
  - 2026-01-26（若涉及江南燃机列）

若**不**把出清价格纳入特征/标签，以上即为最稳的“最大覆盖”主窗。

### 4.3 方案 B：价格增强样本窗

- **目标**：纳入总表事后日前/实时出清价格特征或标签。
- **额外剔除**：  
  - 日前出清无效日：2025-08-12～08-14、**2026-01-01～01-26**  
  - 实时出清无效日：2025-08-08～08-14、2025-08-27～08-31、**2026-01-01～01-26**

价格任务单独维护此窗，不与“仅实际边界特征窗”混用。

### 4.4 方案 C：扩展增强样本窗

- **目标**：把储能实际、煤电实际也加入。
- **起始日**：**2025-12-23**（煤电最晚开始，最干净）。
- **适用**：高质量短窗建模、复杂多特征实验、后验分析。  
- **不适用**：作为最大覆盖训练窗。

## 五、缺失修复与替代规则

### 5.1 修复优先级（统一链路）

**同口径江苏替代 > 同主题邻近字段替代 > 历史模板插补 > 剔除**

### 5.2 按字段类型的修复策略

| 类型 | 策略 | 说明 |
|------|------|------|
| **价格类**（日前/实时出清） | 仅江苏同口径补源；无补源则**整天剔除** | **不建议**用线性插值补价格；补错比缺失更危险 |
| **受电/负荷/风光/燃机** | 江苏同口径替代；单日或少点缺可同日/邻刻插值；整天缺且无补源可用 D-7/D-1 同时刻模板，并打 `quality_flag=template_filled` | **不建议**连续 2 天以上模板硬补、重大节假日用工作日模板 |
| **备用/计划/预测** | 小缺口可同日内插值或前后日同时间模板 | 2025-08-12～08-14 连续缺口：若模型强依赖则**直接剔除样本窗**，不推荐三天整段硬补当高可信数据 |
| **储能/煤电实际** | **绝不对 2025-12-10 前 / 2025-12-23 前的大段 NaN 做补值** | 只能设 `valid_start_date`、提前时段禁用该字段或切晚样本窗 |

### 5.3 建议补全的离散缺日清单（与现有方案一致）

| 日期 | 受影响指标 | 建议处理 |
|------|------------|----------|
| 2025-08-11 | 实际受电、实际负荷、事后江北/江南 燃机/光伏/风力 | 江苏 25年电网边界 同日补全 |
| 2025-08-12～14 | 受电计划、正负备用、日前出清；事前燃机/风光缺行 | 前向填充或标缺失；强依赖则主窗剔除 |
| 2025-11-16、11-23 | 正负备用 | 前向填充或标缺失（无江苏同源） |
| 2026-01-12 | 实际受电 | 江苏 26年电网实际边界 实际受电-华东 补全 |
| 2026-01-26 | 江南燃机 | 前向填充或标缺失 |

## 六、质量标记体系

每个字段每条记录（或至少按日）打质量标签，建议至少分六级：

| 取值 | 含义 | 训练建议 |
|------|------|----------|
| 0 | **original_valid**：原始有效 | 默认使用 |
| 1 | **source_replaced**：江苏替代 | 可用 |
| 2 | **interpolated**：插值修复 | 可用，可监控 |
| 3 | **template_filled**：模板修复 | 谨慎使用或降权 |
| 4 | **structurally_invalid**：结构性无效（如 08-12～14 受电计划整列空） | **过滤**（qflag≥4） |
| 5 | **not_started_yet**：字段尚未开始有效覆盖（储能 2025-12-10 前、煤电 2025-12-23 前） | **过滤** |

模型训练时：**直接过滤 quality_flag ≥ 4**，或把 `quality_flag` 作为辅助输入。

当前实现状态（与脚本一致）：

- **已实现**：`0(original_valid)`、`4(structurally_invalid)`、`5(not_started_yet)`
- **待增强**：`1(source_replaced)`、`2(interpolated)`、`3(template_filled)`

## 七、统一字段编码与分区

### 7.1 推荐 feature_code（与现有 metric_id 对齐）

- 价格：`da_price_south`、`da_price_north`、`rt_price_south_final`、`rt_price_north_final`
- 实际：`import_actual`、`load_actual`、`gas_actual`、`pv_actual`、`wind_actual`、`storage_actual`、`coal_actual`
- 备用：`reserve_pos`、`reserve_neg`
- 计划/预测：`import_plan_boundary`、`import_plan_cleared`、`load_forecast_boundary`、`load_forecast_cleared`

### 7.2 分区与汇总

- 不把“江南+江北+汇总”混成一个字段。拆成：`region=jn`、`region=jb`、`region=all`。
- 总表中“江南+江北=汇总”的字段：raw 层保留分区值；ready 层可派生 `region=all`，并记录 `derived_from = jn + jb`，便于分区/汇总建模与江苏一致性校验。

## 八、ETL 处理流程（五步）

| 步骤 | 内容 | 产出/输出 |
|------|------|-----------|
| 1. 原始抽取 | 从每个文件抽出 日期、时间、区域、字段、值、原始来源元数据，统一转成长表 | Stage 宽表 / 长表 |
| 2. 时间标准化 | 统一生成 `trade_date`、`hh_index`、`ts`；检查每日 96 点、重复、乱序、缺行 | 标准化时间轴 |
| 3. 质量扫描 | 逐字段逐日：row_count、non_null_count、all_null_flag、coverage_start/end | daily_quality_report，识别晚起始、整天 NaN、缺行 |
| 4. 应用修复规则 | 按字段注册表执行：是否允许补、江苏替代字段、是否允许插值、模板阈值 | 带 fill_method、quality_flag 的 L1 表 + 补全记录 |
| 5. 产出 | L1 raw_normalized、daily_quality_report、L2 feature_ready_wide（按样本窗与版本） | 见 二、九 |

当前实现状态（与现脚本对齐）：

- **已实现**：步骤 1、2、5（V0）、以及步骤 3 的基础缺失统计能力（coverage + 缺失明细）
- **部分实现**：步骤 3（标准化 `daily_quality_report` 命名与指标体系待统一）
- **待增强**：步骤 4 自动修复（江苏替代/插值/模板）

## 九、入库版本划分（建议四版，V0 优先）

| 版本 | 字段范围 | 样本窗 | 用途 |
|------|----------|--------|------|
| **V0 主窗版（优先级最高）** | 受电实际、负荷实际、燃机/光伏/风力实际、受电计划、负荷预测、正负备用（可选+日前/实时出清） | **2025-09-01～2025-12-31**（方案 0） | **默认主训练、主回测、主审计**；连续有效、无需处理 8 月与 2026-01 缺日 |
| **V1 基础稳定版** | 同上 A 层主特征 | 方案 A（最大覆盖，剔除 08-11、08-12～14、11-16/23、01-12、01-26 等） | 需要更长覆盖时使用 |
| **V2 价格增强版** | V1 + 日前价格、实时价格 | 方案 B（再剔出清无效日） | 价格预测等任务 |
| **V3 全特征增强版** | V2 + 储能实际、煤电实际 | 方案 C（起点 2025-12-23） | 后期增强实验，非默认底座 |
