## 1. 问题定义与核心难点

### 1.1 任务定义

每个交易日 D-1 的 09:30 之前，需预测 D 日 96 个 15 分钟时段（00:00 至 24:00）的日前出清价格。江南与江北是两个独立的分区，各有独立的出清价序列。

### 1.2 价格分布的特殊结构

与传统时间序列预测不同，江苏电力日前出清价格呈现强烈的**混合分布**特征：

- **地板价聚集**：价格以约 30 元/MWh 的"地板价"高频出现（122 天中，JN 后期折地板价占比超过 50%，JB 更高达 70%）。这是市场出清机制的结果——当供给充裕时，价格不是"低"而是直接跳到规则允许的最低值。
- **正常价格区间**：200-500 元/MWh，呈右偏分布。
- **阶梯突变**：价格在地板价与正常价之间没有平滑过渡，同一天内可能从 30 元跳至 400 元再回 30 元。

这种双峰+阶梯结构意味着：传统的单一回归模型（优化 MSE 或 MAE）在"地板价 vs 正常价"的决策边界上必然产生系统性偏差。

### 1.3 概念漂移

观测期内，地板价出现频率呈**单调递增**趋势。早期折（11 月初）地板价占比约 10-20%，末期折（12 月下旬）地板价占比超过 50%。这意味着：

- 训练集中学到的"地板价是少数事件"的先验，到测试集已不成立。
- 模型必须具备快速适应分布漂移的能力，或在后处理阶段补偿这种漂移。

### 1.4 预测信息的不对称性

D 日的 boundary 曲线（负荷预测、新能源预测等）在 D-1 09:00 可用，但 D-1 日的**实际出力和实时价格**也可用（它们在 D-1 当日实时产生）。这构成一个关键信息源：D-1 的实际运行状况（是否出现大量地板价、实时价格与日前价格的偏差）对 D 日的预测有强相关性。

## 2. 数据与信息约束

### 2.1 可用数据源

| 数据类型 | 时间可用性 | 粒度 | 用途 |
|----------|-----------|------|------|
| D 日 boundary 曲线 | D-1 09:00 | 96 时段 | 供需平衡的先验 |
| D-1（及更早）日前出清价 | D-1 17:00 | 96 时段 | 历史价格模式 |
| D-1 实时出清价 | D 日 00:00 前 | 96 时段 | 前一日市场实际状况 |
| D-1 实际出力/负荷 | D 日 00:00 前 | 96 时段 | 预测误差分析 |

### 2.2 数据规模约束

总数据量约 122 天 × 96 时段 = 11,712 行。在 Expanding Window CV 的早期折中，训练集仅约 60 天 × 96 = 5,760 行。这对特征维度在 100-250 的模型构成了维度灾难的潜在风险，是特征选择和正则化策略的核心动因。

### 2.3 区域独立性

江南和江北作为两个独立的价格区域分别建模，但在特征工程中引入了对方区域的历史价格作为特征（如 JN 模型中使用 JB 的日前价、实时价），因为两区域价格存在跨区相关性（共同受省级供需平衡影响）。

## 3. 特征工程体系

特征工程从 7 个维度系统性地构建了约 250 个候选特征，以下从设计理念角度阐述每个维度。

### 3.1 Boundary 曲线特征（约 60 个）

**设计理念**：Boundary 曲线是 D 日唯一的"未来信息"，直接反映了次日的供需预期。

对 10 个 boundary 列（负荷预测、受电计划、气电计划、光伏预测、风电预测、正/负备用等），每列提取 5 个维度：

| 特征 | 含义 |
|------|------|
| `bnd_{col}` | 当前时段的原始值 |
| `bnd_{col}_diff` | 与前一时段的差分（捕捉日内爬坡率） |
| `bnd_{col}_day_mean` | 当日均值（反映全天水平） |
| `bnd_{col}_day_std` | 当日标准差（反映日内波动） |
| `bnd_{col}_rank` | 当日 96 时段内的分位秩（0-1，反映该时段在全天中的相对位置） |

此外构造三个**衍生特征**：

- **净负荷** `net_load = load - (wind_jb + wind_jn + pv_jb + pv_jn)`：扣除新能源后的残余负荷，是决定火电机组出力（进而决定价格）的核心变量。
- **新能源渗透率** `re_penetration = re_total / load`：新能源占比越高，价格压低越显著（地板价概率越大）。
- **新能源总出力** `re_total` 及其日均值。

### 3.2 历史价格特征（约 40 个）

**设计理念**：日前出清价存在强自相关性（同一 hh_index 的价格在连续几天中高度相关），利用 D-1、D-2、D-7 三个滞后窗口捕捉短期趋势和周周期。

对 4 个价格列（JN/JB 的日前价、节点价），每列在每个滞后窗口上提取：

| 特征 | 含义 |
|------|------|
| `{col}_{lag}_same_hh` | 滞后 d 日的同一时段价格（最直接的 naive 参考） |
| `{col}_{lag}_same_hh_diff` | 同时段价格 - 当日均价（时段内偏离程度） |
| `{col}_{lag}_mean/std/min/max` | 滞后 d 日的日统计量（当日整体价格水平） |

### 3.3 实时价格特征（约 30 个）

**设计理念**：D-1 的实时出清价反映了实际运行中的供需紧张程度，与 D 日的日前价格存在领先-滞后关系。

对 4 个实时价格列（JN/JB 的实时价、最终实时价），在 D-1 和 D-2 窗口上提取：

- 基本统计量（mean/std/min/max/same_hh）
- **地板价频率** `floor_rate = (vals ≤ 50).mean()`：D-1 实时市场中地板价的比例
- **尖峰价频率** `spike_rate = (vals ≥ 400).mean()`：D-1 实时市场中高价的比例

地板价/尖峰频率是关键的**状态指示器**——如果 D-1 实时市场中地板价频繁出现，D 日日前市场大概率也会延续这一模式。

### 3.4 价差特征（约 10 个）

**设计理念**：D-1 日的日前价与实时价之间的价差反映了**预测偏差的方向和幅度**。如果日前价系统性高于实时价（正价差），说明市场对 D-1 的供需预期偏紧，D 日的日前报价可能相应调整。

```
spread = price_dayahead - price_realtime
```

对 JN/JB 分别提取 mean/std/abs_mean/same_hh 四个维度。

### 3.5 预测误差特征（约 10 个）

**设计理念**：D-1 的负荷和新能源出力**预测值**（来自 boundary）与**实际值**之间的误差，反映了预测系统的当前精度状态。如果 D-1 的负荷预测偏高导致日前价偏高，这种"预测乐观"的模式可能延续到 D 日。

```
load_fcst_err = load_forecast - load_actual
re_fcst_err   = re_forecast   - re_actual
```

### 3.6 趋势特征（约 15 个）

**设计理念**：捕捉价格的中期趋势和波动率变化，弥补单日滞后特征无法覆盖的时间尺度。

- **3 日均价趋势** `3d_trend = daily_mean[-1] - daily_mean[0]`：短期方向
- **7 日线性斜率** `trend_7d`：`np.polyfit(x, week_means, 1)[0]`
- **7 日波动率** `vol_7d`：`np.std(week_means)`

### 3.7 地板价结构特征（约 8 个）

**设计理念**：地板价不是随机出现的，它在**特定时段**上有结构性的高发区间（主要是光伏出力高峰时段 09:00-15:00）。通过历史统计揭示每个 hh_index 的地板价倾向。

- **D-1 地板价统计**：`dm1_floor_rate`、`dm1_floor_count`、`dm1_spike_rate`
- **近 7 天同时段地板价频率**：`hh_floor_rate_7d`、`hh_{region}_floor_rate_7d`

### 3.8 日历特征（3 个）

`dow`（星期几 0-6）、`month`、`is_weekend`。工作日/周末和季节性对电力需求模式有系统性影响。

### 3.9 动态特征选择

250 个候选特征中存在大量冗余和噪声。采用**预训练筛选**策略：

1. 使用 CV 的前 `min_train_days`（60 天）作为预训练集、之后 `val_days`（7 天）作为验证集
2. 训练一个 300 树的 LightGBM 回归器（较少的树数以快速评估）
3. 按 `feature_importances_`（split 频次）降序排列
4. 取前 K 个（JN: K=100, JB: K=90）且 importance > 0 的特征

**K 值的确定**：通过网格搜索实验，JN 在 K=100 时 CV MAE 最优，JB 在 K=90 时最优。两者差异来源于区域的数据复杂度不同——JN 价格波动更大，需要更多特征维度来捕捉；JB 结构相对简单，过多特征反而引入噪声。

## 4. 评估框架：Expanding Window 交叉验证

### 4.1 为什么不用 k-fold

时间序列数据存在时间依赖性，k-fold 随机划分会导致"未来信息泄露"——模型可能在训练集中见过测试日之后的价格。Expanding Window CV 严格遵循时间顺序：

```
Fold 0: Train [Day 1..60]   Val [61..67]   Test [68..74]
Fold 1: Train [Day 1..67]   Val [68..74]   Test [75..81]
Fold 2: Train [Day 1..74]   Val [75..81]   Test [82..88]
...
Fold 6: Train [Day 1..109]  Val [110..116] Test [117..123]
```

### 4.2 参数设定

| 参数 | 值 | 选择理由 |
|------|------|---------|
| `min_train_days` | 60 天 | 至少两个月数据确保季节性覆盖 |
| `val_days` | 7 天 | 一周完整周期，用于超参数调优和阈值搜索 |
| `test_days` | 7 天 | 一周完整周期，用于公正评估 |
| `step_days` | 7 天 | 每次窗口前进一周，产生 7 个 fold |

### 4.3 验证集的关键角色

在本框架中，验证集不只是用于 early stopping。它承担了以下调优任务：

- Two-stage 模型的**地板价分类阈值**搜索
- 分位数组合策略的**4 个参数**网格搜索
- 自适应 naive 混合的**混合比例** α 搜索
- 残差修正的**衰减因子** γ 搜索

这意味着验证集的 7 天数据被"用尽"了——在某种程度上，调优过程本身可能导致对验证集的过拟合。这是当前框架的一个已知局限。

### 4.4 评价指标

- **MAE（平均绝对误差）**：主评价指标，对异常值鲁棒，直接反映平均预测偏差
- **RMSE（均方根误差）**：辅助指标，对大偏差更敏感
- **MAPE（平均绝对百分比误差）**：参考指标，但在地板价样本上失真（30 元的地板价即使误差只有 20 元，MAPE 也高达 67%）

### 4.5 Naive 基线

| 基线 | 方法 | JN MAE | JB MAE |
|------|------|-------:|-------:|
| naive_yesterday_same_hh (CV 均值) | 昨日同一时段价格 | 100.43 | 89.99 |
| naive_yesterday_same_hh (全局) | 同上但含 CV 外数据 | 91.40 | 60.37 |
| naive_lastweek_same_hh (全局) | 上周同日同时段 | 135.40 | 110.43 |

JB 的全局 naive 基线（60.37）异常强——这说明 JB 的价格自相关性极强，"照搬昨天"已经是一个很好的策略。模型必须在此基础上提供额外价值。

## 5. 建模策略一：Two-Stage 分类+回归架构

### 5.1 设计动机

单一回归器面对"30 元地板价"和"300 元正常价"的双峰分布时，倾向于预测中间值（如 150 元），两边都不准。Two-stage 架构将问题分解为两个子任务：

1. **分类任务**：这个时段是否会出现地板价？（二分类）
2. **回归任务**：如果不是地板价，价格是多少？（连续值回归）

### 5.2 地板价分类器

地板价的定义为 `y ≤ 50 元/MWh`，分类标签为二值变量。分类器采用 LGBMClassifier，关键设计：

**类别不平衡处理**：早期训练集中地板价占比低（可能仅 10%），采用 `scale_pos_weight` 自动平衡：

```
pos_rate = mean(y_train ≤ 50)
scale_pos_weight = max(1.0, (1 - pos_rate) / pos_rate)
```

当地板价占比为 10% 时，`scale_pos_weight ≈ 9`，即地板价样本的损失权重放大 9 倍。

**分类器超参数**（相对于回归器的差异）：

| 参数 | 分类器 | 回归器 | 差异原因 |
|------|--------|--------|---------|
| n_estimators | 600 | 1500 | 二分类任务更简单 |
| max_depth | 5 | 6 | 限制深度防过拟合 |
| min_child_samples | 30 | 50 | 地板价样本少,需更小的叶节点 |
| early_stopping | 50 轮 | 80 轮 | 更早停止 |

### 5.3 阈值搜索策略

分类器输出概率 `P(floor)`，需要选择一个阈值 `t`：`P(floor) ≥ t` 时预测为地板价。阈值在验证集上搜索，搜索空间为 `t ∈ [0.40, 0.85]`，步长 0.05。

搜索过程有两个约束：

1. **精度约束**：`precision ≥ 0.60`——即分类器预测为地板价的样本中，至少 60% 确实是地板价。这是为了防止 false positive（把正常价格误判为地板价，导致 ~270 元的损失）。
2. **MAE 约束**：将分类器的地板价预测与回归器的非地板价预测组合后的 `combined MAE` 必须低于纯回归器的 MAE。即：只有分类器真的有帮助时才启用。

如果没有任何阈值同时满足这两个约束，阈值设为 1.1（即永远不触发），该折退化为纯回归模型。实验中早期折（地板价样本少）经常出现 `threshold = OFF`，后期折随着地板价增多，分类器逐渐"上线"。

### 5.4 最终预测

```
pred = floor_pred_value  if P(floor) ≥ threshold
       reg.predict(X)    otherwise
```

`floor_pred_value` 默认为 30 元，开启动态模式后取训练集地板价中位数（实验中稳定为 30.0）。

### 5.5 适用性分析

| 区域 | Two-Stage 效果 | 原因 |
|------|---------------|------|
| JN | 有效（V2 MAE=85.51） | 地板价存在但占比适中，分类器可正确覆盖 |
| JB | **非常有效**（V2 MAE=69.95，V5 MAE=69.61） | 地板价占比高且集中，分类器的"硬覆盖"精准匹配 JB 的阶梯分布 |

## 6. 建模策略二：多分位数回归

### 6.1 设计动机

Two-stage 用"是/否"的硬判断来区分地板价和正常价。但价格分布的现实更复杂：一些时段的价格可能在 30 元和 200 元之间摇摆不定，分类器的 0/1 判断无法表达这种不确定性。

分位数回归的核心思想是：不只预测"价格是多少"（点估计），而是预测"价格分布的不同分位点"——从而获得对不确定性的建模能力。

### 6.2 分位数选择

选择 5 个分位数 `α ∈ {0.10, 0.25, 0.50, 0.75, 0.90}`：

- `q10`：价格有 90% 的概率高于此值 → 地板价信号
- `q25`：下四分位 → 低价区间
- `q50`：中位数 → 点预测的主体
- `q75`：上四分位 → 高价区间
- `q90`：价格有 90% 的概率低于此值 → 尖峰信号

### 6.3 LightGBM 分位数回归

与标准回归的唯一区别是损失函数：

- 标准回归：`objective = "regression"`（L2 损失）
- 分位数回归：`objective = "quantile", alpha = α`

分位数损失（pinball loss）的定义：

```
L(y, ŷ, α) = α * max(y - ŷ, 0) + (1 - α) * max(ŷ - y, 0)
```

当 `α = 0.1` 时，低估的惩罚权重是高估的 9 倍，迫使模型预测偏低的值——即分布的第 10 百分位。

### 6.4 分位数组合策略

5 条分位数线不能直接用于预测，需要组合为单一的点预测。组合策略通过 4 个超参数控制，在验证集上网格搜索：

**Step 1: 默认使用 q50（中位数）作为基础预测。**

**Step 2: 地板价自适应覆盖。**

当 `q10 ≤ floor_q10_threshold` 时（即分布的第 10 百分位已经低于阈值），说明模型认为该时段有很高的概率出现地板价。此时用低分位数的加权组合替代 q50：

```
pred = floor_blend_weight × q10 + (1 - floor_blend_weight) × q25
```

这是一种"软覆盖"——不像 Two-stage 那样直接替换为固定值 30 元，而是根据分布形状给出介于 q10 和 q25 之间的值。

**Step 3: 高不确定性区间的 naive 混合。**

当 `q90 - q10 > uncertainty_threshold`（即分布极宽、模型高度不确定）且不处于地板价状态时，引入 naive 预测（昨日同时段价格）作为锚点：

```
pred = (1 - naive_weight) × q50 + naive_weight × naive_yesterday
```

**超参数搜索空间**（在验证集上穷举）：

| 参数 | 搜索范围 | 含义 |
|------|---------|------|
| `floor_q10_threshold` | {50, 80, 100, 130, 160} | q10 低于此值时触发地板价覆盖 |
| `floor_blend_weight` | {0.4, 0.6, 0.8, 1.0} | q10 的权重（1.0 = 完全用 q10） |
| `uncertainty_threshold` | {150, 200, 250, 300, 400} | q90-q10 超过此值视为高不确定性 |
| `naive_weight` | {0.0, 0.2, 0.4} | 高不确定性时 naive 的混合比例 |

搜索空间为 5 × 4 × 5 × 3 = 300 种组合，对 7 天 × 96 时段 = 672 个样本的验证集完全可行。

### 6.5 分位数回归 vs Two-Stage 的本质区别

| 维度 | Two-Stage | 分位数回归 |
|------|-----------|-----------|
| 地板价判断 | 独立分类器，二值硬判断 | 通过 q10 的位置"软判断" |
| 地板价预测值 | 固定值（30 元） | q10 和 q25 的加权平均（更灵活） |
| 不确定性建模 | 无 | 通过 q90-q10 的宽度量化 |
| 参数调优 | 1 个（阈值 t） | 4 个（组合策略参数） |
| 模型数量 | 2（分类器 + 回归器） | 5（每个分位数一个） |
| 训练成本 | 约 2x | 约 5x |

### 6.6 适用性分析

| 区域 | 分位数效果 | 原因 |
|------|-----------|------|
| **JN** | **显著优于 Two-Stage**（79.46 vs 81.51） | JN 价格波动大、双峰分布不对称，分位数的"软判断"更灵活 |
| JB | 显著**劣于** Two-Stage（76.52 vs 69.95） | JB 地板价分布集中，简单的"是/否"硬判断反而更精准，5 个模型的额外复杂性引入了噪声 |

## 7. 建模策略三：分位数+Two-Stage 混合架构

### 7.1 设计动机

分位数回归在 JN 上的优势来自"软覆盖"的灵活性，但它在地板价覆盖率上不如独立分类器——分类器可以用更高精度识别地板价。于是将两者结合：

1. 先用分位数回归产生基础预测（含 q10/q25 的地板价软覆盖）
2. 再用独立的地板价分类器做"显式覆盖"——对分类器高置信度的样本，用固定值替换分位数预测

### 7.2 工作流程

```
Step 1: 训练 5 个分位数回归模型 → q10, q25, q50, q75, q90
Step 2: 在验证集上调优组合参数 → 得到分位数组合预测
Step 3: 训练地板价分类器 → 输出 P(floor)
Step 4: 在验证集上搜索阈值 t，基准预测为 Step 2 的组合结果
Step 5: 对测试集，先用分位数组合，再对 P(floor) ≥ t 的样本覆盖为地板价值
```

关键的创新点：**Step 4 的 MAE 对比基准是分位数组合预测，而非纯回归预测**。只有分类器在分位数基础上还能进一步降低 MAE 时，才启用覆盖。

### 7.3 实验结果

这是 JN 上的**最优架构**（V4h MAE=78.27），较纯分位数（79.46）再降 1.5%，较纯 Two-Stage（81.51）降 4.0%。

对 JB 则无帮助——分位数基础本身就不如 Two-Stage，在此基础上叠加分类器无法挽回。

## 8. 后处理技术群

在基座模型（Two-Stage 或分位数混合）产生初始预测后，本研究探索了四种后处理技术，它们作用于 CV 的每一折，独立于基座模型。

### 8.1 时间衰减样本加权

**原理**：面对概念漂移（地板价频率递增），赋予近期样本更高的训练权重，使模型更快适应最新分布。

**公式**：

```
weight_i = 2^((date_i - date_max) / half_life_days)
```

- 最新样本权重 = 1.0
- `half_life_days` 天前的样本权重 = 0.5
- `2 × half_life_days` 天前的样本权重 = 0.25
- 以此类推

权重直接传入 LightGBM 的 `sample_weight` 参数，影响所有模型（分类器、回归器、分位数回归器）。

**实验结论**：

| 区域 | 效果 | 分析 |
|------|------|------|
| JN | 轻微负面（MAE +1.9%） | 分位数回归本身已通过逐折的 4 参数调优适应了分布变化；衰减反而削弱了早期"正常价格"样本的影响力 |
| JB | 轻微正面（MAE -0.3%） | Two-stage 的单回归器对分布漂移更敏感，衰减帮助它更快学习增多的地板价模式 |

**启示**：时间衰减的价值取决于基座模型的适应性。自适应能力强的模型（如分位数回归）不需要额外的衰减；适应性弱的模型（如单回归器）从中受益。

### 8.2 自适应 Model-Naive 混合

**原理**：在市场相对平稳的时期，"照搬昨天"（naive）的策略可能比复杂模型更准确。该技术在验证集上动态调优模型与 naive 的混合比例：

```
final_pred = (1 - α) × model_pred + α × naive_yesterday
```

α 在 `[0.00, 0.55]` 范围内以 0.05 步长搜索，选择使验证集 MAE 最小的值。

**核心机制——"自动退化"**：

| 验证集情况 | 调优结果 | 含义 |
|-----------|---------|------|
| 模型远优于 naive | α = 0.00 | 完全依赖模型 |
| 模型与 naive 相当 | α = 0.10~0.30 | 适度混合，降低方差 |
| naive 明显更好 | α = 0.50 | 大比例采用 naive |

**逐折 α 值的典型模式**：

| Fold | 测试期 | α (JN) | α (JB) | 市场状态 |
|-----:|--------|-------:|-------:|---------|
| 0 | 11月初 | 0.25 | 0.10 | 平稳→适度混合 |
| 1 | 11月中 | 0.05 | 0.00 | 模型表现好 |
| 2 | 11月下 | 0.50 | 0.50 | 高度平稳→大量混合 |
| 3 | 12月初 | 0.20 | 0.05 | 市场转变，模型略优 |
| 4-6 | 12月中后 | 0.00 | 0.00-0.05 | 突变加剧→完全依赖模型 |

**与时间衰减的交互**：两者功能部分重叠——都是为了应对分布变化。实验表明同时开启时效果不如单独使用，在 JN 上最优组合是 ANB + DFV（不用衰减），在 JB 上最优组合是全部三项（衰减 + ANB + DFV）。

### 8.3 动态地板价填充值

**原理**：Two-stage 和分位数混合模型在判定为地板价时需要一个**填充值**。默认固定为 30 元，但如果未来地板价的实际水平发生变化（如市场规则调整导致地板价变为 20 元），固定值将产生系统性偏差。

动态模式取训练集中所有实际地板价样本（y ≤ 50）的中位数作为填充值。在当前数据中，该中位数稳定为 30.0 元——即效果中性，但提高了对未来变化的鲁棒性。

### 8.4 后处理技术效果综合对比

以下表格展示了在各自最优基座模型上叠加后处理的消融实验：

#### 江南（基座: V4h 分位数+Two-Stage 混合）

|-----------|----:|-------:|-----:|
| 无后处理 (V4h) | 78.27 | — | 118.52 |
| + 时间衰减(30d) + DFV | 79.76 | +1.9% | 120.21 |
| + ANB + DFV（**JN 最优**） | **78.20** | **-0.1%** | 117.85 |

#### 江北（基座: V3 Two-Stage）

|-----------|----:|-------:|-----:|

## 9. 负面实验：分时段分组建模

### 9.1 假设

96 个时段的价格行为存在结构性差异。通过对每个 hh_index 的价格统计特征（均价、标准差、地板价率、尖峰率、变异系数）进行 KMeans 聚类（k=3），识别出三个自然群体：

| 组别 | 时段 | hh_index | 特征 |
|------|------|----------|------|
| A_solar（光伏谷段） | 09:00-15:00 | 37-60 | 高地板率(21-30%), 高波动(CV 0.55-0.65) |
| B_evening（傍晚峰段） | 15:00-23:00 | 61-92 | 低地板率(6-10%), 低波动(CV 0.28-0.35) |
| C_night（夜间/早间） | 23:00-09:00 | 1-36, 93-96 | 中等地板率(12-19%), 中波动(CV 0.44-0.48) |

假设为每个组训练独立模型可以让模型更专注于该组的分布特征，避免组间干扰。

### 9.2 实现方案

- 按 hh_index 范围将数据集切分为 2 或 3 个子集
- 每个子集独立运行完整的 expanding_window_cv（含特征选择、模型训练、后处理）
- 对较小的分组自动缩放参数：
  - `feature_select_top_k = max(30, base_k × n_hh / 96)`——防止小分组的特征维度过高
  - `min_child_samples = max(原值, 50 × 96 / n_hh)`——小分组使用更保守的正则化

### 9.3 结果

| 方案 | JN MAE | vs 统一 | JB MAE | vs 统一 |
|------|-------:|-------:|-------:|-------:|
| 统一模型 (最优) | **78.20** | — | **69.61** | — |
| 3 组 (A/B/C) | 84.17 | +7.6% | 73.42 | +5.5% |
| 3 组 v2 (缩放参数) | 87.48 | +11.9% | 73.83 | +6.1% |
| 2 组 (B独立, AC合并) | 84.64 | +8.2% | 74.03 | +6.3% |

### 9.4 失败分析

分组后各组独立 MAE 揭示了根本原因：

| 组别 | JN MAE | JB MAE | 样本量/折 |
|------|-------:|-------:|---------:|
| A_solar | **122.8** | **125.4** | ~24 hh × 7 天 = 168 |
| B_evening | 67.3 | 53.4 | ~32 hh × 7 天 = 224 |
| C_night | 74.5 | 58.2 | ~40 hh × 7 天 = 280 |

**A_solar 组是最难的时段（光伏高峰导致地板价频发），同时也是样本量最少的组。** 分组后每折训练集仅约 24 × 60 = 1440 行（且早期地板价样本极少），对 30-100 个特征的 LightGBM 来说严重不足，导致过拟合或欠拟合。

**统一模型为什么更好**：LightGBM 作为树模型，具备天然的**内部分段能力**——它可以用 `hh_index` 作为分裂变量，在树的内部隐式实现"如果 hh_index ∈ [37,60] 则走左子树"。这等价于一种"软分组"，但同时享受全量 11,712 行样本的统计量。

**与统计学习理论的关系**：分组建模本质上是增加了模型的方差（每个组用更少的数据拟合更灵活的模型），在数据不足时方差增大超过了偏差减少，导致泛化性能下降。这是经典的偏差-方差权衡。

## 10. 负面实验：残差自回归修正

### 10.1 假设

模型在验证集上的残差具有**per-hh_index 的系统性偏差**——如果模型在验证集的 hh_index=42 上持续低估 20 元，那么在测试集的 hh_index=42 上大概率也会低估，可以用验证集的偏差来修正测试集的预测。

### 10.2 方法

1. 在 CV 的每一折，计算验证集上每个 hh_index 的平均残差偏差：

```
bias[hh] = mean(y_val - pred_val)  对每个 hh_index
```

2. 用衰减因子 `γ` 控制修正幅度：

```
pred_test_corrected = pred_test + γ × bias[hh_test]
```

3. `γ ∈ [0.0, 1.0]` 在验证集上搜索最优值（注意：这里用验证集自身的残差来搜索 γ，存在一定程度的信息泄露，但 γ 只有 11 个候选值，过拟合风险可控）。

### 10.3 结果

| 配置 | JN MAE | JN RMSE | JB MAE | JB RMSE |
|------|-------:|--------:|-------:|--------:|
| V5 最优 (无残差) | **78.20** | 117.85 | **69.61** | 110.36 |
| + 残差+ANB+DFV | 79.29 | **116.37** | 69.95 | **109.35** |
| + 残差+DFV (无ANB) | 79.15 | 116.15 | 70.29 | 110.01 |

### 10.4 分析

**MAE 中性偏负，RMSE 显著改善**——这是一个有意义的分化信号：

- **MAE 无改善**说明：7 天验证集上学到的 per-hh 偏差在接下来 7 天的测试集上**不稳定**。概念漂移导致偏差模式快速变化（一周前的偏差方向可能一周后已经逆转）。
- **RMSE 改善 1-1.3%** 说明：残差修正虽然在"平均"意义上没有帮助，但它成功压缩了**极端大偏差**。对于那些偏差方向恰好一致的时段，修正有效；对于偏差方向逆转的时段，γ 的衰减控制了损失。

**γ 的时间演变趋势**验证了概念漂移假说：

| 时段 | 典型 γ | 解释 |
|------|------:|------|
| 早期折 (11月) | 0.7-1.0 | 市场平稳，偏差模式稳定 |
| 中期折 (12月初) | 0.3-0.5 | 地板价增多，偏差模式开始不稳定 |
| 后期折 (12月中后) | 0.1-0.4 | 概念漂移加剧，γ 自动降低 |

γ 的自适应调节机制工作正常，但这恰恰证明了残差修正的前提——"偏差模式跨期稳定"——在本问题中不成立。

### 10.5 与自适应 naive 混合的功能重叠

两者都是利用验证集信息做后处理调整。当同时开启时，naive 混合已经从方向上修正了模型与市场趋势之间的偏差，残差修正再进一步修正的边际空间很小，两者部分抵消。

### 11.2 关键发现

**1. 江南和江北需要不同的建模策略。**

这不是偶然——两个区域的价格分布特征存在本质差异：

| 特征 | JN | JB |
|------|-----|-----|
| 价格波动性 | 高（CV ≈ 0.45-0.65） | 更高（CV ≈ 0.48-0.63） |
| 地板价集中度 | 地板价值分散 | 地板价值集中于 30 附近 |
| 最优架构 | 分位数+混合 | 纯 Two-Stage |
| 最优后处理 | ANB + DFV | 衰减 + ANB + DFV |

JB 的地板价行为更加"干净"——要么是 30，要么不是——这恰好匹配分类器的硬判断。JN 的地板价区域更加模糊，分位数回归的连续输出更适合。

**2. 后处理技术的改善幅度有限但方向明确。**

从 V4h/V3 到 V5 最优，后处理仅带来 0.1-0.5% 的 MAE 改善。这说明：

- 基座模型（特征工程 + 架构选择）决定了 95% 以上的性能
- 后处理是"最后一公里"优化，收益递减但方向稳定
- 后处理的真正价值在于**鲁棒性**——在不同的市场状态下自适应调整（如 α 在市场平稳时增大、突变时归零）

**3. 特征工程是最大的改善杠杆。**

V2 → V3 通过引入实时价格、价差、预测误差、趋势等特征，JN MAE 从 85.51 降至 81.51（-4.7%）。这一改善超过了之后所有模型架构和后处理优化的总和。

**4. 数据量是当前的核心瓶颈。**

所有"增加模型复杂度"的尝试（分组建模、残差修正）都受限于 122 天的数据量。分组建模在样本量减半后立即退化；残差修正在 7 天验证集上学到的偏差模式无法跨期泛化。这不是方法本身的问题，而是数据规模的约束。

**5. MAE 与 RMSE 存在有意义的分化。**

多种后处理技术（特别是残差修正）在 MAE 上中性但 RMSE 上有显著改善。这说明它们有效压缩了极端误差但未能改善中等误差。在不同的业务目标下（追求"平均准"还是"避免大错"），最优配置不同：

| 目标 | JN 最优 | JB 最优 |
|------|---------|---------|
| 最小化 MAE | V5 (ANB+DFV) = 78.20 | V5 (全部) = 69.61 |
| 最小化 RMSE | V7 (残差+DFV) = 116.15 | V7 (残差+ANB+DFV) = 109.35 |

## 12. 结论与展望

### 12.1 当前最优配置

| 区域 | 架构 | MAE | vs Naive | 关键命令参数 |
|------|------|----:|--------:|------------|
| JN | 分位数+Two-Stage 混合 + ANB + DFV | 78.20 | -22.1% | `--quantile-two-stage --adaptive-naive-blend --dynamic-floor-value` |
| JB | Two-Stage + 衰减(30d) + ANB + DFV | 69.61 | -22.6% | `--two-stage --time-decay 30 --adaptive-naive-blend --dynamic-floor-value` |

### 12.2 技术路线总结

```
特征工程 (V2→V3: -4.7% JN)
    ├── 最大改善杠杆
    └── boundary + 历史价 + 实时价 + 价差 + 预测误差 + 趋势 + 地板价结构

模型架构选择 (V3→V4h: -4.0% JN)
    ├── JN: 分位数+Two-Stage 混合（灵活的不确定性建模）
    └── JB: 纯 Two-Stage（精准的地板价硬覆盖）

后处理微调 (V4h→V5: -0.1% JN, -0.5% JB)
    ├── 自适应 Naive 混合（自动退化机制）
    ├── 时间衰减（对弱适应性模型有效）
    └── 动态地板价填充（鲁棒性保障）

负面实验
    ├── 分组建模（数据量不足 → 方差爆炸）
    └── 残差修正（概念漂移 → 偏差不稳定，但 RMSE 改善）
```

### 12.3 后续方向

1. **引入气象特征**：温度、辐照量等可以直接影响负荷和光伏出力预测的精度，有望进一步改善特征工程层面的表现。
2. **集成多版本模型**：将 JN 的 V4h 与 V3 加权平均，或对 JB 的 V2 与 V5 进行集成，利用不同架构在不同市场状态下的互补性。
3. **数据积累后重新评估**：当数据积累到 200+ 天时，重试分组建模和残差修正，这两种方法的效果直接受数据量制约。
4. **在线学习**：当部署到生产环境后，可以使用真实的 D-1 预测误差作为 D 日的输入特征（目前 CV 框架中未利用历史折的残差信息作为特征）。
